# モンスターハンターワイルズ ダメージ計算アプリ 設計書

- 作成日: 2026-05-12
- 対象: モンスターハンターワイルズ（**Wilds専用**、他シリーズ作品は対象外）
- ステータス: ドラフト → ユーザーレビュー待ち

---

## 1. 概要

モンスターハンターワイルズにおいて、装備・武器・スキル・装飾品・護石・モーションパターンの割合から
**期待ダメージ** と **立ち回り期待DPS** を算出するWebアプリを開発する。

### 主な特徴

- 全14武器種対応
- 物理 / 属性ダメージの正確な計算
- 装飾品・護石（鑑定護石含む）への対応
- 条件付きスキル（挑戦者・逆襲など）の発動時間割合をユーザーが調整可能
- 立ち回りパターンを複数登録し、それぞれの割合でDPSを加重平均
- 抜刀術が乗るモーションを個別フラグ管理
- 計算ロジックを別タブで日本語数式として表示
- ビルドと結果を **画像(PNG) / テキスト / JSON** で出力
- JSONペーストで他人のビルドをインポート
- **4テーマ対応**（Ember / Aurora / Forest / Solar）でユーザーの好みに合わせた配色切替

---

## 2. 目的とスコープ

### 含むもの

- ハンター側の全要素（武器倍率・会心・斬れ味・スキル・装飾品・護石・バフアイテム）
- モンスター側（部位肉質・属性肉質・全体防御率・怒り状態）
- 立ち回りのモーション割合に基づくDPS計算
- 条件付きスキルの発動時間割合
- Wilds固有機構（傷つけ・傷口・集中攻撃・相殺・操虫棍の弱点集中等）
- ビルドのローカル保存・JSON/テキスト/画像での共有
- PWA対応（オフライン動作・インストール可能）

### 含まないもの

- 他のモンハンシリーズ作品の計算式
- マルチプレイ環境のシミュレーション（味方バフ共有等）
- 装備の自動最適化（オートビルダー）
- サーバーを介したビルド共有（外部DB保存）

### 4.2 レイヤ分離の原則

- **`calc/`** はUIから完全に独立した純粋関数群。Reactにも依存させない。
  - 入力：`CalcInput`（プレーンなオブジェクト）
  - 出力：`DamageResult`（プレーンなオブジェクト）
  - 副作用なし → 単体テストが容易
- **`data/`** はJSONを読み込んで型付きオブジェクトに変換するだけのレイヤ。
- **`store/`** はZustandでUI状態とユーザー入力を管理。
- **`components/`** はstoreとdataから情報を取り出し、`calc/`を呼び出して結果を描画。

## 6. 計算ロジック

### 6.1 計算フロー（近接武器の場合）

#### STEP 1 ─ 攻撃力期待値の合成

```
攻撃力期待値 = 武器攻撃力
            + 攻撃スキル補正
            + アイテムバフ補正  （鬼人薬・粉塵など）
            + Σ(条件付き攻撃スキル補正 × 発動時間割合)
```

#### STEP 2 ─ 会心率期待値

```
会心率期待値 = 武器会心率
            + 見切りスキル補正
            + 弱点特効補正        （対象肉質≧45のときのみ）
            + 抜刀術【技】補正    （抜刀モーションのみ）
            + Σ(条件付き会心スキル補正 × 発動時間割合)
　→ -1.00 〜 +1.00 にクランプ
```

#### STEP 3 ─ 会心期待係数

```
会心期待係数 = 1 + 会心率期待値 × (会心倍率 - 1)

  会心倍率の決まり方:
    通常時 = 1.25
    超会心Lv3発動時 = 1.40
    マイナス会心時 = 0.75
```

#### STEP 4 ─ 物理ダメージ（モーション単位）

```
物理ダメージ = 攻撃力期待値 ÷ 武器係数
           × (モーション値 ÷ 100)
           × 斬れ味補正(物理)
           × 会心期待係数
           × 武器固有補正        （鬼人化・練気色など）
           × (物理肉質 ÷ 100)
```

抜刀術【力】の補正は `isDraw=true` のモーションだけ攻撃力期待値に加算する。

#### STEP 5 ─ 属性ダメージ（モーション単位）

```
属性ダメージ = 武器属性値
           × 属性スキル補正       （属性攻撃強化など）
           × 斬れ味補正(属性)     （基準は緑、補正幅は物理より小さい）
           × 属性会心期待係数     （会心撃【属性】発動時 ×1.35）
           × (属性肉質 ÷ 100)
```

#### STEP 6 ─ パターンの総ダメージとフレーム

```
パターン総ダメージ = floor( (Σ物理ダメージ + Σ属性ダメージ) × 全体防御率 )
パターン総フレーム = Σ モーションフレーム数
```

物理と属性は**合算してから**切り捨てる（個別に切り捨てない）。

#### STEP 7 ─ 立ち回り期待DPS

```
立ち回り期待DPS = Σ(パターン総ダメージ × パターン割合)
              ÷ Σ(パターン総フレーム × パターン割合)
              × 60

  ※ 60 = 1秒あたりフレーム数
  ※ パターン割合の合計は1.0（100%）になる必要がある
```

### 6.2 遠距離武器（ライトボウガン・ヘビィボウガン・弓）の差分

```
遠距離物理 = 攻撃力期待値 ÷ 武器係数
           × (弾威力 ÷ 100)         ※ モーション値の代わり
           × 飛距離補正              （クリティカル距離で最大）
           × 速射補正                （LBGの速射弾のみ）
           × 会心期待係数
           × 武器固有補正            （弓のビン効果など）
           × (物理肉質 ÷ 100)

  ※ 斬れ味補正は適用しない
  ※ 代わりに飛距離補正で威力が増減
```

### 6.3 Wilds固有機構の扱い

| 機構 | 計算上の扱い |
|------|------------|
| **傷つけ・傷口** | 対象部位に「傷口」フラグを持たせ、傷口時のhitzoneを別管理（傷口時は肉質+補正） |
| **集中攻撃** | 1つのモーションパターンとして登録可能。武器種固有の専用モーション値を持つ |
| **相殺攻撃** | モーションパターンの一種として扱う |
| **操虫棍：弱点集中** | 武器固有補正で対応：弱点ヒット時に ×1.75 |
| **狩猟笛：音波ダメージ** | 斬れ味・会心を無視した定数ダメージ計算パスを別途用意 |
| **ガンランス：砲撃** | 攻撃力非依存の定数ダメージ。砲撃専用関数で計算 |
| **チャージアックス：榴弾ビン爆発** | 攻撃力非依存の定数ダメージ |

### 6.4 主要スキルの補正適用方式

| スキル | 適用方式 |
|--------|---------|
| 攻撃 | 攻撃力期待値に加算 |
| 見切り | 会心率期待値に加算 |
| 超会心 | 会心倍率を 1.25 → 1.40 に置換 |
| 弱点特効 | 対象部位の物理肉質≧45 のときのみ会心率に加算 |
| 会心撃【属性】 | 属性会心期待係数を適用（会心時 ×1.35） |
| 抜刀術【力】 | `isDraw=true` のモーションのみ攻撃力期待値に加算 |
| 抜刀術【技】 | `isDraw=true` のモーションのみ会心率期待値に加算 |
| 業物 | 斬れ味の消費抑制（モデル上は平均斬れ味を選択させる） |
| 挑戦者 | 条件付きスキル。攻撃+/会心+ を発動時間割合で線形補間 |
| 逆襲 | 条件付きスキル。攻撃+ を発動時間割合で線形補間 |
| 属性攻撃強化 | 武器属性値に乗算 |

## 7. データ層

```
public/data/

```

データ源はWildsコミュニティのオープンデータを利用。
更新時は `scripts/` 配下のスクリプトで取得・整形してJSONを差し替える。

### 7.2 ユーザーデータ（IndexedDB / Dexie.js）

```typescript
interface Build {
  id: string
  name: string
  weaponId: string
  armor: {
    head?:  { id: string; decorations: (string | null)[] }
    chest?: { id: string; decorations: (string | null)[] }
    arms?:  { id: string; decorations: (string | null)[] }
    waist?: { id: string; decorations: (string | null)[] }
    legs?:  { id: string; decorations: (string | null)[] }
  }
  talisman: 
    | { type: 'preset'; id: string; decorations: (string | null)[] }
    | { type: 'custom'; skills: { name: string; level: number }[]; decorations: (string | null)[] }
  conditionalSkills: { name: string; uptime: number }[]   // 0.0〜1.0
  motionPatterns: MotionPattern[]
  buffs: string[]
  target: { monsterId: string; partId: string; enraged: boolean; wounded: boolean }
  createdAt: number
  updatedAt: number
}

interface MotionPattern {
  name: string
  motions: {
    motionName: string
    motionValue: number
    frames: number
    isDraw: boolean
  }[]
  ratio: number    // 0.0〜1.0、合計が1.0になるようバリデーション
}
```

## 9. テスト戦略

- 「装備選択 → 計算 → 出力」のE2Eシナリオ（最小限）

## 10. 段階的な実装方針

実装計画は別途 `plans/` 配下に記載するが、大まかには次の順序：

1. プロジェクト初期化（Vite + React + TS + Tailwind + Zustand + Dexie + PWA）
2. **デザイントークン整備**（OKLCHベースのCSS変数、4テーマ、Geist/JetBrains Mono/Noto Sans JP導入）
3. 型定義と最小のゲームデータJSON（太刀1武器・防具数点・モンスター1体）で骨組み
4. `calc/melee.ts` の基本式実装と単体テスト
5. 共通UIプリミティブ実装（Header、Card、Field、Chip、Slider、Toggle、StatTile など）
6. 装備ビルダーUIの骨組み（武器・防具・装飾品・護石・スキル表示）
7. 立ち回りパターンUI と条件付きスキルUI
8. 計算結果表示と内訳パネル（ヒーローDPS + tween + breakdown bar）
9. 計算式・ロジック解説タブ
10. テーマ選択タブと themeStore（永続化込み）
11. JSON/テキスト/PNG 入出力
12. 全14武器種への対応拡張（特に遠距離・特殊機構）
13. PWA設定とリリース

## 11. リスクと未確定事項

| 項目 | 内容 | 緩和策 |
|------|------|-------|
| 計算式の精度 | 検証用に参照した3サイトのうち2サイトから情報取得できなかった | 実装後にユーザー側で実機値との突き合わせを行う。テストケースは順次追加 |
| ゲームデータの提供元 | コミュニティデータの公開状況に依存 | データ更新スクリプトを `scripts/` に分離し、いつでも差し替え可能にする |
| Wilds固有機構の式 | 傷口時の補正値や集中攻撃の正確なMVが未確定の可能性 | プレースホルダで実装し、ユーザーが上書き可能なフィールドも用意する |
| 操虫棍・狩猟笛・ガンランス砲撃 | 武器固有計算の網羅性 | `calc/weapons/` で武器ごとにモジュール化し、テストで網羅 |

## 12. 用語集

- **武器係数**: 表示攻撃力を実数値に変換するための除数。武器種ごとに異なる
- **モーション値(MV)**: 各攻撃モーション固有の威力値
- **斬れ味補正**: 紫1.39 / 白1.32 / 青1.20 / 緑1.05 / 黄1.00 / 橙0.75 / 赤0.50（物理）
- **肉質**: モンスター部位ごとの耐久度。45以上が「弱点」扱い
- **発動時間割合**: 条件付きスキルが発動している時間の割合（0.0〜1.0）
- **全体防御率**: クエストごとに設定されたダメージ補正
- **傷口**: Wilds新機構。攻撃で部位に傷ができ、対象として強化攻撃可能
